# GCP-AWS Federation: Deployment & Installation Guide

This document explains how to host the `get-gcp-token` APT repository on Google Cloud Storage (GCS) and how users can install it on their systems.

## 1. Hosting the Repository on GCS

Follow these steps to upload the repository generated by the `generate_repo.sh` script to a GCS bucket.

### Prerequisites
- A Google Cloud project with the `gsutil` or `gcloud storage` CLI configured.
- A GCS bucket (e.g., `gs://my-apt-repo`).

### Upload Steps

1. **Create and Configure Bucket (One-time)**:
   Ensure the bucket is publicly readable if you want the repository to be accessible without authentication.
   ```bash
   gsutil mb gs://my-apt-repo
   gsutil iam ch allUsers:objectViewer gs://my-apt-repo
   ```

2. **Sync Repository Files**:
   Upload the `gcs-repo` directory to the root of your bucket.
   ```bash
   gsutil -m rsync -r gcs-repo gs://my-apt-repo
   ```

## 2. Client Installation Instructions

Provide these instructions to users who need to install `get-gcp-token` on their Linux instances (Ubuntu/Debian).

### Step 1: Trust the Repository GPG Key
Download and add the public key to the system's keyring so `apt` can verify the package signatures.
```bash
curl -fsSL https://storage.googleapis.com/my-apt-repo/public.key | sudo gpg --dearmor -o /usr/share/keyrings/gcp-aws-federation.gpg
```

### Step 2: Add the Repository to Sources
Add the GCS bucket as a software source.
```bash
echo "deb [signed-by=/usr/share/keyrings/gcp-aws-federation.gpg] https://storage.googleapis.com/my-apt-repo stable main" | sudo tee /etc/apt/sources.list.d/gcp-aws-federation.list
```

### Step 3: Install the Package
Update the package index and install the binary.
```bash
sudo apt update
sudo apt install get-gcp-token
```

---

## 3. Post-Installation Verification

Once installed, users can verify the installation using the following commands:

- **Check Binary**:
  ```bash
  which get_gcp_token
  ```
- **Access Documentation**:
  ```bash
  man get_gcp_token
  ```
- **Run Help**:
  ```bash
  get_gcp_token --help
  ```

---

## Maintenance: Updating the Package

To release a new version:
1. Update the `VERSION` variable in `build_packages.sh`.
2. Run `./build_packages.sh`.
3. Run `./generate_repo.sh`.
4. Re-sync with GCS: `gsutil -m rsync -r gcs-repo gs://my-apt-repo`.
